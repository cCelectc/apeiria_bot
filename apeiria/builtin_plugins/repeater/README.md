# 群聊复读机

_✨ Apeiria 面向群聊氛围的概率复读插件 ✨_

</div>

## 📖 介绍

这是 Apeiria 的内置基础功能插件，用于在指定群聊中按连续复读氛围原样复读一次。

适合这些场景：

- 群友连续发送同一文本、表情或稳定图片时，希望机器人偶尔参与复读
- 主人希望复读行为只在指定平台和群范围内生效
- 群聊娱乐功能需要保持低噪音，并能通过概率曲线避免机械触发

插件默认随框架加载，但默认使用空白名单，因此不会在任何群实际复读。运行状态只保存在内存中，机器人重启或插件重载后会清空当前复读轮次。

## 💿 安装

<details open>
<summary>作为 Apeiria 内置插件使用</summary>

群聊复读机为 Apeiria 内置插件，默认随框架加载，无需单独安装。

如果你关闭了此插件，可在项目插件配置中重新启用：

    apeiria.builtin_plugins.repeater

</details>

## ⚙️ 配置

在 Apeiria 的插件配置中可调整以下配置项：

| 配置项 | 必填 | 默认值 | 说明 |
|:---:|:---:|:---:|:---:|
| `repeat_threshold` | 否 | `2` | 连续不同用户发送相同内容达到该次数后开始抽取复读概率 |
| `platforms` | 否 | `["qq"]` | 启用的平台标识列表 |
| `group_mode` | 否 | `allowlist` | 群范围模式，可选 `allowlist` 或 `blocklist` |
| `allow_groups` | 否 | `[]` | 允许生效的群，格式为 `platform:group_id` |
| `deny_groups` | 否 | `[]` | 禁止生效的群，格式为 `platform:group_id`，冲突时优先 |
| `base_probability` | 否 | `0.25` | 刚达到阈值时的复读概率 |
| `max_probability` | 否 | `0.9` | 连续复读人数增加后概率的上限 |
| `saturation_extra` | 否 | `4` | 从基础概率爬升到概率上限所需的额外重复次数 |
| `debug` | 否 | `false` | 启用后记录跳过原因和调试日志 |

示例：

```yaml
repeater:
  repeat_threshold: 2
  platforms:
    - qq
  group_mode: allowlist
  allow_groups:
    - qq:123456
  deny_groups: []
  base_probability: 0.25
  max_probability: 0.9
  saturation_extra: 4
  debug: false
```

## 🎉 使用

### 行为表

| 行为 | 权限 | 需要@ | 范围 | 说明 |
|:---:|:---:|:---:|:---:|:---:|
| 群友连续发送相同内容 | 公开 | 否 | 配置群聊 | 达到阈值后按概率原样复读一次 |
| 以 `/` 开头的消息 | 公开 | 否 | 配置群聊 | 视为命令消息并打断当前复读轮次 |

支持的内容包括精确文本、带稳定 ID 的表情 / face 段，以及带稳定文件 ID 的图片段。混合消息按原始顺序比较。

## 注意

- 默认 `allow_groups` 为空，不会影响任何群。
- 同一用户连续发送相同内容不会累计触发次数，但会保留当前轮次。
- Bot 自己发送的消息会被忽略，不会累计或重置轮次。
- 不支持回复、转发、卡片、无稳定 ID 图片等内容；这些消息会打断当前轮次。
- 同一轮复读一旦尝试发送，无论发送成功还是失败，都不会再次触发，直到内容变化。
- 只有 `/` 前缀会被视为命令消息并打断轮次，其他前缀按普通文本参与比较。
- 插件不接触 AI 层、对话记录、数据库或持久化状态。
